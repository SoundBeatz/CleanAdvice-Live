# Clean Admin blueprint

Clean Admin is the future backoffice for managing Clean360 OS without editing code.

## Purpose

Allow Clean360 to manage knowledge, services, machines, products, rules and workflows from a private admin interface.

## First admin sections

- Machines
- Service protocols
- Floor types
- Services
- Quote rules
- Products
- Chemicals
- Videos
- Users and roles

## Machine editor

Fields:

- Brand
- Model
- Category
- Description
- Applications
- Parts
- Symptoms
- Service protocols
- Compatible pads
- Compatible chemicals
- Manuals
- Videos

## Service protocol editor

Fields:

- Protocol ID
- Symptom
- First question
- Diagnostic questions
- Checkpoints
- Likely causes
- Standard answer
- Escalation rules
- Required ticket fields

## Rule editor

Fields:

- Rule ID
- Trigger
- Conditions
- Action
- Priority
- Safety level
- Active yes/no

## Workflow editor

Fields:

- Workflow ID
- Trigger
- Steps
- Required data
- Output
- Escalation

## Permissions

Planned roles:

- Owner
- Admin
- Service manager
- Sales manager
- Content editor
- Viewer

## MVP note

The first MVP can use Markdown and schema files. Clean Admin comes after the first working video-agent and service/quote flows.
